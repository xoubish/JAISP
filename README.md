# JAISP (Joint AI Survey Processing)

## Overview

JAISP is a multi-instrument foundation model stack for Rubin + Euclid imaging.

The workflow has two layers:

1. Foundation pretraining (`JAISPFoundationV6` stable, `JAISPFoundationV7` experimental): self-supervised masked band reconstruction.
2. Downstream heads: detection, astrometry concordance, and PSF/forced photometry.

The central idea is to learn one spatially precise shared representation, then attach task-specific heads that reuse it.

## Foundation Architectures

### Stable: v6 (`models/jaisp_foundation_v6.py`)

`JAISPFoundationV6` is a dense masked autoencoder trained on Rubin + Euclid tiles.

Key design choices:

- No patch tokenization. Full-resolution convolutional processing keeps sub-pixel spatial detail.
- Per-band stems (`BandStem`) with GroupNorm. Each band has its own noise/PSF front-end.
- Mean aggregation in noise-normalized space across context bands.
- ConvNeXt downsampling backbone + transformer bottleneck at `H/8 x W/8`.
- U-Net decoder with FiLM conditioning by target band identity.
- InformationMap-weighted L1 loss to focus learning on informative source pixels.

### Curriculum

- Phase A (`cross_instrument_prob=0.0`): Rubin-only masked band prediction.
- Phase B (`cross_instrument_prob=1.0`): Rubin + Euclid masked band prediction.
- Euclid inputs are downsampled to Rubin resolution (512x512) in Phase B.
- Tiles without Euclid coverage automatically fall back to Rubin-only behavior.

This Phase B training is the key bridge that makes downstream Rubin/Euclid alignment easier.

### Experimental: v7 mixed resolution (`models/jaisp_foundation_v7.py`)

`JAISPFoundationV7` keeps early Rubin, VIS, and NISP processing at native resolution, then fuses streams on a shared latent physical grid.

Key design choices:

- Rubin, VIS, and NISP each have their own native-resolution branch depth before fusion.
- Cross-stream fusion happens after resizing latent features to a common physical scale, not by resizing raw inputs to one image grid.
- The decoder is target-aware and reconstructs back to the held-out band's native resolution.
- The objective is still InformationMap-weighted L1 in noise-normalized units, so the learning signal stays close to `v6`.

This is the architecture to try when we want to preserve more Euclid VIS detail for downstream astrometry and detection.

## Downstream Heads

### 1) Detection (`models/detection/`)

`JaispDetector` is a DETR-style set predictor on top of the v6 encoder bottleneck.

- Input: multi-band tile dict (`images`, `rms`) in JAISP format.
- Backbone: frozen `JAISPEncoderV6` by default (optional end-to-end fine-tune).
- Decoder: learned object queries + transformer decoder.
- Outputs per query:
  - normalized centroid `(x, y)`
  - class logits (`star`, `galaxy`, `artifact`)
  - objectness logit
  - log-flux proxy
- Training: Hungarian assignment + DETR-style matched/unmatched losses.

Current label source:

- Pseudo-labels are generated by classical Rubin detection (`g+r+i` detection image) and concentration-based star/galaxy split.
- `artifact` is currently an output class but not separately pseudo-labeled.

See `models/detection/README.md` for details.

### 2) Astrometry concordance (`models/astrometry2/`)

`V6AstrometryMatcher` reuses frozen Phase B stems for Rubin and VIS encoders, predicts local offsets with uncertainty, and solves a smooth distortion field.

Key idea:

- Learn local Rubin<->VIS offset proposals from image morphology.
- Weight anchors by predicted uncertainty.
- Fit a smooth field (grid or NN solver) in sky-coordinate offsets (`dRA*`, `dDec`).

See `models/astrometry2/README.md`.

### 3) PSF + forced photometry (`models/photometry/`)

Photometry uses a separate PSF model and vectorized forced flux estimator.

- `PSFNet`: spatially varying per-band PSF model (base Gaussian + learned residual in log-PSF space).
- `TilePhotometryPipeline`: precompute PSF grid, interpolate PSFs at source positions, extract stamps, subtract local background, run matched filter.
- `matched_filter`: Cramer-Rao optimal linear flux estimator under known PSF + per-pixel variance.

See `models/photometry/README.md`.

## Quick Start

### Foundation training

```bash
# Phase A (Rubin only)
python models/train_jaisp_foundation_v6.py \
  --rubin_dir data/rubin_tiles_ecdfs \
  --euclid_dir data/euclid_tiles_ecdfs \
  --output_dir models/checkpoints/jaisp_v6 \
  --cross_instrument_prob 0.0 \
  --epochs 60

# Phase B (Rubin + Euclid, resume from Phase A)
python models/train_jaisp_foundation_v6.py \
  --rubin_dir data/rubin_tiles_ecdfs \
  --euclid_dir data/euclid_tiles_ecdfs \
  --output_dir models/checkpoints/jaisp_v6_phaseB \
  --resume models/checkpoints/jaisp_v6/checkpoint_best.pt \
  --cross_instrument_prob 1.0 \
  --epochs 120

# Experimental v7 mixed-resolution training
python models/train_jaisp_foundation_v7.py \
  --rubin_dir data/rubin_tiles_ecdfs \
  --euclid_dir data/euclid_tiles_ecdfs \
  --output_dir models/checkpoints/jaisp_v7 \
  --cross_instrument_prob 1.0 \
  --epochs 80
```

### Detection head training

```bash
python models/detection/train_detection.py \
  --rubin_dir data/rubin_tiles_ecdfs \
  --euclid_dir data/euclid_tiles_ecdfs \
  --encoder_ckpt models/checkpoints/jaisp_v6_phaseB/checkpoint_best.pt \
  --out models/checkpoints/detector_v1.pt \
  --epochs 50
```

### Astrometry matcher training

```bash
python models/astrometry2/train_astro_v6.py \
  --v6-checkpoint models/checkpoints/jaisp_v6_phaseB/checkpoint_best.pt \
  --rubin_dir data/rubin_tiles_ecdfs \
  --euclid_dir data/euclid_tiles_ecdfs
```

### PSF training

```bash
python models/photometry/train_psf_net.py \
  --rubin_dir data/rubin_tiles_ecdfs \
  --euclid_dir data/euclid_tiles_ecdfs \
  --out models/checkpoints/psf_net_v1.pt \
  --epochs 20
```

## Important Current Status Notes

- `v6` remains the stable default foundation path.
- `v7` is an experimental mixed-resolution path intended to test whether retaining native VIS structure improves downstream astrometry and detection.
- Detection labels are pseudo-labels (not a curated catalog), so detection metrics should be interpreted as bootstrap quality, not final science quality.
- The detection module is new and intended as a foundation-aligned head that can later swap pseudo-labels for survey catalogs.
- This repository is an active research codebase. Architecture and training defaults evolve as new experiments land.
